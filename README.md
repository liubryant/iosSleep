# iosSleep

一款面向 iOS 的睡眠监测应用。核心目标是在不打扰用户睡眠的前提下，通过夜间声音识别、睡眠记录、健康数据整合和趋势分析，帮助用户了解打鼾、咳嗽、说梦话、磨牙和环境噪音等睡眠事件。

完整开发计划见 [PLAN.md](PLAN.md)。当前工程已包含 SwiftUI 三 Tab 应用骨架、Tide 声音资源、睡眠监测服务、HealthKit 接口和“我的”模块基础页面。

## 产品定位

iosSleep 参考主流睡眠与助眠应用的成熟体验：睡前提供安静、低负担的声音入口；夜间自动记录关键声音事件；醒来后给出清晰的睡眠报告；长期维度呈现趋势变化。产品重点不是做复杂医疗诊断，而是提供个人睡眠习惯观察、异常提醒和可回溯记录。

核心体验：

- 睡前：选择助眠声音、设置睡眠监测、确认麦克风和健康权限。
- 夜间：本地采集音频，进行声音事件识别，只保存必要片段和统计结果。
- 晨间：展示睡眠时长、声音事件时间线、噪音强度、关键片段和趋势摘要。
- 长期：按日、周、月查看打鼾/咳嗽/噪音等指标变化，并与 HealthKit 睡眠数据对齐。

## 首页框架

应用首页采用底部 3 Tab：

1. 声音
2. 睡眠
3. 我的

### 声音

声音模块负责助眠内容和夜间声音环境管理，类似小睡眠、潮汐这类应用中的自然声/白噪音入口。

主要功能：

- 自然声场景：雨声、海浪、森林、风扇、溪流、火焰、咖啡馆等。
- 混音播放：支持 2-4 个声音叠加，分别调整音量。
- 定时关闭：15/30/45/60/90 分钟或入睡后渐弱。
- 收藏与最近播放：减少睡前操作成本。
- 声音场景详情：封面、简介、推荐使用场景。
- 与睡眠监测联动：开始监测后可继续播放助眠声音，并在录音识别中降低自家播放声音对事件判断的影响。

### 睡眠

睡眠模块是核心监测与报告入口。

主要功能：

- 开始睡眠：一键进入夜间监测模式。
- 实时状态：显示正在监测、麦克风状态、环境音量、剩余电量提醒。
- 睡眠报告：展示本次睡眠总览、声音事件、噪音水平、时间线和趋势。
- 事件分类：
  - 打鼾
  - 咳嗽
  - 说梦话
  - 磨牙
  - 环境噪音
- 事件时间线：按夜间时间轴展示声音事件分布。
- 声音片段回放：只保留被识别为事件的短片段，用户可手动删除。
- Swift Charts 图表：
  - 睡眠时长柱状图
  - 打鼾/咳嗽事件次数趋势
  - 环境噪音分贝曲线
  - 夜间事件密度热力图
- HealthKit 集成：
  - 读取系统睡眠分析数据
  - 写入应用记录的睡眠时段
  - 对齐 Apple Watch 或系统睡眠数据

### 我的

我的模块负责个人设置、权限、隐私和数据管理。

主要功能：

- 用户资料：昵称、头像、睡眠目标。
- 权限管理：麦克风、HealthKit、通知。
- 监测设置：
  - 事件灵敏度
  - 是否保存声音片段
  - 片段保留天数
  - 仅 Wi-Fi 备份
  - 低电量暂停提醒
- 数据管理：
  - 导出睡眠报告
  - 删除本地录音片段
  - 清空全部历史数据
- 隐私说明：明确音频默认本地处理，用户授权后才保存或导出。
- 关于应用：版本、反馈、隐私政策、用户协议。

## 技术链路

核心声音识别链路：

```text
麦克风
↓
PCM 音频
↓
频谱图 Mel Spectrogram
↓
AI 模型
↓
打鼾 / 咳嗽 / 说梦话 / 磨牙 / 环境噪音
```

推荐技术栈：

- SwiftUI：主界面与导航框架。
- AVFoundation：
  - AVAudioRecorder 用于长时间录音落盘。
  - AVAudioEngine 用于实时 PCM 流处理。
  - AVAudioSession 管理后台录音、播放和中断恢复。
- Core ML：
  - YAMNet 作为通用声音识别基座。
  - 后续可微调或叠加自定义分类器，提升打鼾、磨牙、梦话等睡眠场景准确率。
- SoundAnalysis：
  - 可作为 Core ML 声音分类的实时分析封装。
- Accelerate：
  - FFT、窗函数、Mel 滤波器组等频谱计算。
- HealthKit：
  - 读取/写入睡眠分析数据。
  - 与系统健康数据形成统一睡眠视图。
- Swift Charts：
  - 睡眠趋势、声音事件、噪音曲线等数据可视化。
- SwiftData/Core Data：
  - 存储睡眠记录、事件索引、片段路径、统计结果。

## 声音识别设计

### 音频采集

夜间监测时使用 AVAudioSession 的 record/playAndRecord 模式：

- 采样率：16 kHz 或 44.1 kHz，视模型输入要求统一重采样。
- 声道：单声道优先。
- 分片窗口：建议 0.96s-2s 一帧，滑动窗口分析。
- 后台能力：开启 Audio Background Mode，保证锁屏后持续监测。
- 中断恢复：处理来电、闹钟、耳机切换、系统音频中断。

### 特征提取

YAMNet 通常以 log Mel Spectrogram 作为输入。流程：

1. PCM 归一化。
2. 分帧和加窗。
3. FFT 得到频谱。
4. Mel 滤波器组映射。
5. log 压缩。
6. 输入 Core ML 模型。

### 分类策略

YAMNet 是通用声音模型，原始标签可能不完全等同业务标签。应用层需要做标签映射和二次规则：

- Snoring / Breathing -> 打鼾
- Cough -> 咳嗽
- Speech / Conversation / Whispering -> 说梦话候选
- Toothbrush / Chewing / Clicking 等需结合频率特征，作为磨牙候选
- Traffic / Rain / Wind / Domestic noise / Music -> 环境噪音

为了减少误判：

- 使用连续多帧投票，而不是单帧直接判定。
- 设置事件冷却时间，避免同一声音被切成大量重复事件。
- 结合音量阈值和频段特征过滤低置信度结果。
- 对用户播放的助眠声音建立白名单或回声抑制策略。
- 允许用户在报告中标记误判，用于后续本地规则优化。

## 隐私与权限

睡眠监测会使用麦克风，必须把隐私设计放在第一层。

原则：

- 默认本地处理音频，不上传原始录音。
- 默认只保存识别到的短声音片段，可在设置中关闭。
- 用户可以随时删除单条片段、单次睡眠记录或全部数据。
- 首次使用前明确说明麦克风用途。
- HealthKit 读写权限分开申请，只请求睡眠相关数据。

需要配置：

- `NSMicrophoneUsageDescription`
- `NSHealthShareUsageDescription`
- `NSHealthUpdateUsageDescription`
- Background Modes: Audio

## 开发里程碑

### M1：项目骨架

- 创建 SwiftUI App。
- 搭建 3 Tab：首页框架。
- 完成基础主题、导航、空状态。
- 接入本地声音资源列表和基础播放器。

### M2：录音与本地存储

- 接入麦克风权限。
- 实现夜间录音 Session。
- 保存 SleepSession 和基础统计。
- 支持开始、暂停、结束监测。

### M3：声音识别 MVP

- 集成 YAMNet Core ML 模型。
- 实现 PCM -> Mel Spectrogram -> 模型推理。
- 输出基础分类和置信度。
- 生成 SleepEvent。

### M4：睡眠报告

- 使用 Swift Charts 展示事件时间线、次数趋势、噪音曲线。
- 支持事件详情和声音片段回放。
- 支持删除片段。

### M5：HealthKit

- 申请 HealthKit 权限。
- 读取系统睡眠数据。
- 写入应用睡眠记录。
- 报告中展示系统睡眠和声音事件的对齐结果。

### M6：打磨与上线准备

- 优化后台稳定性和耗电。
- 完善误判过滤和灵敏度设置。
- 完成隐私政策、权限说明和 App Store 文案。
- 增加本地数据导出和清理策略。

## 风险点

- 后台长时间录音可能带来耗电和系统中断问题，需要重点测试。
- YAMNet 是通用模型，磨牙和说梦话的准确率可能需要自定义数据集或后处理规则增强。
- 助眠声音播放和麦克风识别同时进行时，可能造成模型误判，需要做播放源过滤。
- HealthKit 睡眠数据的读写能力受用户授权和系统数据源影响，报告需要处理数据缺失。
- 保存音频片段涉及隐私敏感数据，必须提供明确开关和删除能力。

## 当前结论

iosSleep 的第一版应优先完成“可开始监测、可识别声音事件、可生成睡眠报告”的闭环。声音模块作为睡前入口提升使用频率，HealthKit 和长期趋势作为第二阶段增强。技术上以 SwiftUI + AVFoundation + Core ML + HealthKit + Swift Charts 为主线，保证核心链路全部在本地完成。
